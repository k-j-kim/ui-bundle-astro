## Prerequisites

- **Node 22+**
- **Salesforce CLI** (`sf`) — authenticated to a target org
- An existing SFDX project (has a `sfdx-project.json`)

## Scaffold

<Steps>

1. **Generate the bundle.**

   ```sh
   sf template generate ui-bundle --name MyApp
   ```

   Creates `force-app/main/default/uiBundles/MyApp/`.

2. **Install dependencies.**

   ```sh
   cd force-app/main/default/uiBundles/MyApp
   npm install
   ```

3. **Start the dev server.**

   ```sh
   npm run dev
   ```

   Open `http://localhost:4321`. Astro hot-reloads on every save.

</Steps>

## Build

<Tabs>
  <TabItem label="npm">
    ```sh
    npm run build
    ```
  </TabItem>
  <TabItem label="pnpm">
    ```sh
    pnpm build
    ```
  </TabItem>
</Tabs>

This runs `astro build` then `scripts/postbuild.mjs`. The post-build step:

1. Rewrites absolute `/assets/...` URLs to relative paths (required because Salesforce serves the bundle under a dynamic mount path).
2. Externalizes inline `<script>` tags to separate `.js` files (Salesforce's strict CSP blocks inline scripts).
3. Validates that `dist/index.html` exists for SPA fallback routing.

## Deploy

From your SFDX project root:

```sh
sf project deploy start \
  --source-dir force-app/main/default/uiBundles \
  --target-org my-org
```

<Aside type="tip">
Run `sf project deploy validate` first to dry-run the deployment without activating it.
</Aside>

## What you get

After scaffolding, the dev server shows a dashboard with:

- **Dashboard** (`/`) — stat tiles and quick links
- **Accounts** (`/accounts`) — live data table querying Account records via GraphQL
- **Contacts** (`/contacts`) — same pattern for Contact records
- **Opportunities** (`/opportunities`) — with currency formatting
- **About** (`/about`) — tech stack summary

All data pages use the same `RecordList` React island, configured per-page with different columns. See [React islands](/guides/islands/) for how this works.
