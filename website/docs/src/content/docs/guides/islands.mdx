Most of the template is static Astro components (zero JS). When you need interactivity — search, filtering, live data — you use a React island.

## How it works

Write a React component in a `.tsx` file, import it in an `.astro` page, and add a `client:*` directive to hydrate it in the browser:

```astro
---
// src/pages/accounts.astro
import AppLayout from '@layouts/AppLayout.astro';
import Card from '@components/Card.astro';
import RecordList from '@components/RecordList.tsx';
---
<AppLayout title="Accounts" eyebrow="Sales" heading="Accounts">
  <Card title="All accounts" subtitle="Ordered by name">
    <RecordList
      client:load
      sobject="Account"
      columns={[
        { label: 'Name',     field: 'Name' },
        { label: 'Industry', field: 'Industry', muted: true },
        { label: 'Type',     field: 'Type',     muted: true },
        { label: 'Phone',    field: 'Phone',    muted: true, align: 'right' },
      ]}
    />
  </Card>
</AppLayout>
```

Without `client:load`, the component renders to static HTML at build time and never hydrates. With it, Astro ships the React runtime and component JS only for that island.

## Hydration directives

| Directive | When it hydrates |
|---|---|
| `client:load` | Immediately on page load |
| `client:idle` | After the page is idle (`requestIdleCallback`) |
| `client:visible` | When the element enters the viewport |
| `client:media="(max-width: 768px)"` | When a media query matches |
| `client:only="react"` | Client-only, no SSR — skips build-time rendering |

Use `client:load` for above-the-fold interactive content. Use `client:visible` for components further down the page to defer JS loading.

## Example: `RecordList` component

This is the main React island in the template. It queries Salesforce records via the UI API GraphQL endpoint and renders a searchable table:

```tsx
// src/components/RecordList.tsx
import { useEffect, useMemo, useState } from 'react';
import { runQuery } from '@lib/sdk';

export interface RecordListColumn {
  label: string;
  field: string;
  align?: 'left' | 'right';
  currency?: boolean;
  muted?: boolean;
}

export interface RecordListProps {
  sobject: string;
  columns: RecordListColumn[];
  orderBy?: string;
  orderDir?: 'ASC' | 'DESC';
  first?: number;
}

export default function RecordList({
  sobject,
  columns,
  orderBy = 'Name',
  orderDir = 'ASC',
  first = 25,
}: RecordListProps) {
  const [rows, setRows] = useState<any[] | null>(null);
  const [error, setError] = useState<string | null>(null);
  const [query, setQuery] = useState('');

  // Build GraphQL query from column spec
  const gql = useMemo(() => {
    const fields = columns.map(c => `${c.field} { value }`).join(' ');
    return `
      query ${sobject}List {
        uiapi {
          query {
            ${sobject}(
              first: ${first},
              orderBy: { ${orderBy}: { order: ${orderDir} } }
            ) {
              edges { node { Id ${fields} } }
            }
          }
        }
      }
    `;
  }, [sobject, columns, orderBy, orderDir, first]);

  useEffect(() => {
    let cancelled = false;
    (async () => {
      try {
        const res = await runQuery(gql);
        if (cancelled) return;
        const edges = res?.data?.uiapi?.query?.[sobject]?.edges ?? [];
        setRows(edges.map((e: any) => {
          const node = e.node;
          const out: any = { Id: node.Id };
          for (const c of columns)
            out[c.field] = node?.[c.field]?.value ?? null;
          return out;
        }));
      } catch (e: any) {
        if (!cancelled) setError(e?.message ?? String(e));
      }
    })();
    return () => { cancelled = true; };
  }, [gql]);

  // ... render table with search filtering
}
```

The key pattern: the GraphQL query is built dynamically from the `columns` prop, so the same component powers Accounts, Contacts, and Opportunities with zero code duplication.

## Reusing `RecordList` for different objects

Each page passes different props:

```astro
// src/pages/contacts.astro — Contact records
<RecordList
  client:load
  sobject="Contact"
  columns={[
    { label: 'Name',  field: 'Name' },
    { label: 'Title', field: 'Title', muted: true },
    { label: 'Email', field: 'Email', muted: true },
    { label: 'Phone', field: 'Phone', muted: true },
  ]}
  orderBy="LastName"
/>
```

```astro
// src/pages/opportunities.astro — with currency formatting
<RecordList
  client:load
  sobject="Opportunity"
  columns={[
    { label: 'Name',   field: 'Name' },
    { label: 'Stage',  field: 'StageName', muted: true },
    { label: 'Close',  field: 'CloseDate', muted: true },
    { label: 'Amount', field: 'Amount', align: 'right', currency: true },
  ]}
  orderBy="CloseDate"
  orderDir="DESC"
/>
```

<Aside type="tip">
The `currency: true` column option formats numbers as USD. Extend `formatValue()` in `RecordList.tsx` to support other currencies.
</Aside>

## Writing your own island

Any `.tsx` file with a default export works:

```tsx
// src/components/Counter.tsx
import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);
  return (
    <button onClick={() => setCount(c => c + 1)}>
      Clicked {count} times
    </button>
  );
}
```

```astro
---
import Counter from '@components/Counter.tsx';
---
<Counter client:load />
```

Without `client:load`, `Counter` renders its initial state as static HTML and never becomes interactive.
